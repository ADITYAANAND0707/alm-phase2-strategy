11. Model Adaptation Strategy

The strategy recommends efficient fine-tuning on the accepted MAL dataset as the first serious model-adaptation step — not full-parameter retraining. The accepted registry is the prerequisite, not the experiment target.

DimensionFull-parameter fine-tuningEfficient fine-tuning
What it updatesAll model weightsSmall trainable subset — adapters or LoRA-style methods
StrengthHighest adaptation capacityFaster, cheaper, easier to iterate
WeaknessExpensive and riskier with evolving dataLower adaptation ceiling than full retraining
Best fit for current Phase 2Not firstRecommended first step
Planning directionEfficient fine-tuning should only begin once the accepted dataset has measurable quality. The accepted registry is the prerequisite — not the experiment target.

12. Research Methods Reference

These research methods informed the Phase 2 pipeline design. Core methods directly shape how data is generated and filtered. Supporting methods provide evidence and precedent for scaling and quality decisions.

Method / Reference Main Idea Best Use for MAL Difficulty Role
Self-Instruct Seed to generate to filter bootstrapping loop Base MAL synthetic pipeline — seeds drive generation, validation filters output Medium Core
Evol-Instruct / WizardLM Grow instruction complexity through progressive rewriting Harder modify, policy-edge, and repair examples from simpler seeds Medium Core
Orca Explanation-rich synthetic supervision using chain-of-thought traces Explain and repair pairs — richer MAL task behavior data Medium Core
AgentInstruct Agentic prompt/response creation from raw document sources Scaled MAL data-factory workflows — future high-volume generation High Core reference
Phi-4 Data-quality-centric training — curated synthetic data beats more raw data Evidence for the curated-data-first strategy in Phase 2 Low Supporting
SmolLM workflow Practical small-model fine-tuning using synthetic data pipelines MAL-specific efficient fine-tuning experiments (LoRA-scale) Low Supporting
Scale Labs / Hybrid Combining human expert data with model-generated data at scale Scaling signal for when Phase 2 outgrows manual seeding Medium Supporting
Planning directionCore references (Self-Instruct, Evol-Instruct, Orca) should drive the design of the seed-to-registry pipeline. Supporting references inform scaling and quality decisions but should not dilute the MAL-specific focus.
