11. Model Adaptation Strategy

Efficient fine-tuning on the accepted MAL dataset is the recommended first adaptation step — not full-parameter retraining. The registry is the prerequisite.

Not First — Full-parameter Fine-tuning

All Weights Updated

  • Updates every parameter in the base model
  • Highest adaptation capacity ceiling
  • Expensive — GPU memory, time, infrastructure cost
  • Riskier with evolving datasets — full re-runs required on data changes
  • Suitable when dataset is stable, large, and final
Not the first step for Phase 2
Recommended — Efficient Fine-tuning (LoRA)

Small Trainable Subset

  • Adapters or LoRA-style methods — adds small trainable matrices
  • Base model weights stay frozen — only adapters update
  • Faster to iterate as dataset grows sprint by sprint
  • Cheaper — runs on single GPU, fits current team scale
  • Lower adaptation ceiling than full retraining — acceptable trade-off
Start here once registry has measurable quality
Planning directionStart efficient fine-tuning only once the accepted dataset has measurable quality. The accepted registry is the prerequisite — not the experiment target.

12. Research Methods Reference

Methods that directly informed the Phase 2 pipeline design. Core methods shape how data is generated and filtered. Supporting methods provide scaling and quality evidence.

Core Methods — directly shape the pipeline
Self-Instruct
Seed → generate → filter bootstrapping loop for instruction data
Base MAL synthetic pipeline — seeds drive generation, Quality Gate filters output
Core · Medium difficulty
Evol-Instruct / WizardLM
Grow instruction complexity through progressive in-context rewriting
Harder modify, policy-edge, and repair MAL examples evolved from simpler seeds
Core · Medium difficulty
Orca
Explanation-rich synthetic supervision using chain-of-thought traces
Explain and repair task pairs — richer, reasoning-grounded MAL task behavior data
Core · Medium difficulty
AgentInstruct
Agentic prompt/response generation from raw document sources at scale
Scaled MAL data-factory workflows — future high-volume automated generation
Core reference · High difficulty
Supporting Methods — scaling & quality evidence
Phi-4
Data-quality-centric training — curated synthetic data beats more raw data
Evidence for the curated-data-first strategy that anchors Phase 2
Supporting · Low complexity
SmolLM Workflow
Practical small-model fine-tuning using synthetic data pipelines
MAL-specific efficient fine-tuning experiments (LoRA-scale, single GPU)
Supporting · Low complexity
Scale Labs / Hybrid
Combining human expert data with model-generated data at scale
Scaling signal for when Phase 2 outgrows manual seeding and annotation
Supporting · Medium complexity
Planning directionCore references (Self-Instruct, Evol-Instruct, Orca) drive the seed-to-registry pipeline design. Supporting references inform scaling decisions but should not dilute the MAL-specific focus.
