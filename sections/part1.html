1. Executive Summary

The ALM platform generates, validates, and manages enterprise application models written in MAL — a purpose-built domain language. It already handles user intent, live validation, and versioned persistence. What it lacks is an offline data factory: a repeatable pipeline that creates high-quality training and evaluation examples at scale.

Phase 2 builds that factory in three stages. First, a Jinja2-based template engine produces controlled MAL variants from manually authored "gold seeds" — the safest, most explainable starting point. Second, LLM-driven expansion adds diversity once the template baseline is solid. Third, a human review layer promotes accepted records into a registry that feeds prompting, evaluation, and eventually fine-tuning.

  • Scope: MAL only. Phase 2 generates examples for application, entity, workflow, policy, agent, integration, event, and evolution. UI constructs are excluded from the first data wave.
  • Stage 2 integrated. The Jinja2 pipeline is built and merged into the shared team repo — 56 canonical gold seeds (Divya’s data/seed_scenarios/telecom_v1.jsonl), 4 grammar-compliant templates, a telecom domain pack, 336 candidate records generated, and a React/Vite frontend dashboard active in mock mode.
  • LLM expansion comes later. Introduce it only after validation gates, graph-compile checks, and registry discipline are in place. Do not skip the template baseline.
  • This document is a living report. Update the progress tracker, decision log, coverage grid, and example library as Phase 2 matures.

4. Scope Definition

Phase 2 synthetic-data work is MAL-specific and tightly bounded. This document defines what is in scope, what is excluded, and what the objective is — so the team stays aligned as the pipeline grows.

Confirmed from repoCurrent ALM code supports MAL generation, deterministic intent parsing, validation, graph handling, vector retrieval, and versioned storage.
Planning directionThe pipeline design defines five stages: gold seeds, Jinja2 expansion, LLM expansion, validation/filtering, and human review.
Proposed for Phase 2The offline synthetic-data factory is new work to be added outside the existing runtime flow.
Phase 2 Scope RuleDecision
Primary language scopeMAL only
Supported construct familiesapplication, entity, workflow, policy, agent, integration, event, evolution
Excluded from first data waveUI constructs such as screens, components, actions, and navigation
Primary objectiveProduce validator-approved user-intent to MAL examples for generation quality, specification support, and later model adaptation
