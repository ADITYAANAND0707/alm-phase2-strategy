Quick Brief — What is this and where are we?
We are building an offline data factory for MAL — the domain language used by the ALM platform. The factory takes manually written seed examples, expands them into hundreds of validated variants using Jinja2 templates, then uses LLMs to add diversity, and finally routes everything through human review into an accepted dataset registry. The registry feeds model prompting, evaluation benchmarks, and eventually fine-tuning. Stage 2 (Jinja2 expansion) is built and integrated into the shared team repo with 56 canonical seeds, 336 candidates, and a live frontend dashboard. See Section 7b for the full user flow — every page, every button, and what each step produces. Stage 3 (validation routing) is next.
